Requirements

R1. Relational Database Schema & Indexes (Supabase/PostgreSQL)

Define a complete, production-ready SQL script (schema.sql) to set up the core database tables (user_profiles, recipes, recipe_ingredients, and recipe_steps). The schema must enforce strict relational integrity via foreign keys and cascade deletes to avoid orphaned data. It must use strict NUMERIC(10,1) decimal types for ingredient quantities to prevent scaling errors. It must also implement Generalized Inverted Indexes (GIN) on array fields (cube_tags and dietary_tags) to support sub-100ms recipe filtering.

R2. Row Level Security (RLS) Policies

Write SQL scripts inside schema.sql to configure and enforce Row Level Security (RLS) on all database tables. The policies must prevent users from accessing or modifying other users' profiles, while allowing public read access to published recipes, step metadata, and ingredients. It must define admin/service_role bypasses for CMS operations.
